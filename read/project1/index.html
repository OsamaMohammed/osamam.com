    Private Project 1 - Huawei LTE 4G UPCC EDR Server

    by Osamah Saadallah M. Mon Nov 12, 2018

    Big Database Processing Client/Server Application for Huawei LTE 4G UPCC EDR

    Two applications were developed to help process and maintain data generated from Huawei UPCC called EDR (Enhanced Data Record).

    Problem:

    Time taken to search for one user inside at least 9.8 GB files per day (UPCC generated files) would take 30-50 minutes, and another 15 minutes to clean and finalize the results in a readable form.

    Data received from UPCC are huge (~9.9 GB/day).

    Solution:

    Developing a server that listen for new files and instantly processes it, encode it in a smaller data form.

    Developing a client for reading the generated files from the server in high speed.

    Server Development:

    A server were developed to catch newly received files from Huawei UPCC (EDR files) decode it, and transform it into a smaller, more efficient form for lower data size and high performance in searching.

    Server Results:

    Files received per day (~9.9 GB) takes only 484 MB and it can be compressed up to 55 MB for archiving using LZMA2 compression method.

    Client Development:

    Developing a client to read the processed data in high speed and present the data in a readable format (CSV File).

    Client Result:

    Data for one user in a day can be obtained in less than 10 seconds!. A sample result file can be found here

    Conclusion:

    Time taken to search and read a user data is now ~10 seconds compared to 30-50 minutes is a huge success.

    Data size is now takes almost 4.7% of the original data, and 0.05% with compression!.
